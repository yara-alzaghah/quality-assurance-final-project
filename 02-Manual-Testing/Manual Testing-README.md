# Part 02 – Manual Testing & Defect Reporting

## Overview
This folder contains the **manual testing deliverables** for the SauceDemo web application, prepared as part of the Software Testing – Quality Assurance capstone assignment.

The objective of this phase is to design, execute, and document **manual test cases**, capture execution evidence, and report identified defects in a clear and professional manner.

---

## Contents of This Folder

### 1. SauceDemo Test Cases.xlsx
This file contains the **manual test cases** designed based on the approved test scenarios from Part 01.

Each test case includes:
- Test Case ID and Title
- Preconditions and test data
- Step-by-step execution steps
- Expected results
- Actual results
- Test execution status
- Execution evidence reference

The test cases cover:
- Login and authentication
- Product listing and cart management
- Checkout workflow and form validation
- Logout functionality
- Happy path and negative scenarios

---

### 2. SauceDemo Bug Reports.xlsx
This file contains the **defect log** created during manual test execution.

Each defect includes:
- Defect ID and title
- Affected module
- Environment details
- Preconditions and test data
- Steps to reproduce
- Expected vs actual result
- Severity and priority
- Supporting evidence

Defects were logged for both functional issues and validation-related findings, following a structured defect reporting format.

---

## System Under Test
- **Application:** SauceDemo (https://www.saucedemo.com)
- **Type:** Web-based e-commerce demo application
- **Focus Areas:** Login, products, cart, checkout, and logout
